1	Introduction
This Document refers the Standalone Build Process. Standalone Endpoint is adherence to Group Information & Cyber Security Standard: Standalone Machine Management.
2	Objective
The objective of this document is to describe the process to be followed to create Standalone Build for bank.  
3	Risks
The Standalone Machine Management Standard mandates that adequate controls are implemented and that the Machines are protected or secured to a level equivalent to the Machines connected to the Group network. Failure to adopt and implement this Information Security Standard may expose the Machines to both data leakage and to malware resulting in:
•	Loss of customer trust
•	Adverse publicity and reputational damage
•	Financial loss
•	Regulatory fines
4	Technical controls

This section defines the security controls mandatory on the Standalone Computer. Kindly adhere to below technical control

4.1	Standalone Machine with SOE Build 

•	On the SOE Built Machine, Open Admin Command Prompt
•	Uninstall Password scrambler
•	Reset the Local Admin Password
•	Disjoin the machine from SCB domain.
•	Delete the NAC / All the SCB Certificate using certlm.msc command.
•	Ensure that Standalone Machines are licensed, vendor-supported, and not obsolete.
 



•	Ensure that relevant patches and software updates are applied during delivery of PC and monthly.
*If the Standalone machine does not have Internet access, kindly download and install the required patches offline from official website.
•	Enable both Audit and Event Logging.
Goto -> gpedit.msc
Configure below setting to enable audit log
 

•	Disable media auto-run features.
Goto -> gpedit.msc
Configure below setting to Disable media auto-run
Computer Configuration\ Policies\ Administrative Templates\Windows Components\AutoPlay Policies
Turn off Autoplay	Disabled
Prevent AutoPlay from remembering user choices	Disabled
Set the default behaviour for Autorun	Disabled
Turn off Autoplay	Disabled


 


•	Enable screen time-out and lock [15 minutes].
 

 

 


•	Create User specific access accounts. No shared or generic accounts are allowed for use.
•	Create administrator specific privileged user access accounts for support team. No shared or generic accounts are allowed for use.
Run -> Compmgmt.msc
 
•	Apply password complexity for both user and privileged accounts as per Group ICS Standards.

Computer Configuration\Policies\Windows Settings\Security Settings\Account Policies\Password
Enforce password history	12 passwords remembered
Maximum password age	90 days
Minimum password age	1 days
Minimum password length	8 characters
Password must meet complexity requirements	Enabled
Store passwords using reversible encryption	Disabled

 
 



•	Ensure that network access and allowed traffic (to and from the Standalone Machine) is defined and controlled, with the approach and its deployment is reviewed at least every 6 months.
•	Default Bitlocker encryption to be turned off and enable Bitlocker encryption locally.  Support team to retrieve and safely store the recovery keys for future use.
•	Ensure only encrypted and authorized Portable Storage Devices are used for data transfers.
•	Ensure that the machine/user certificates are deleted before handing them to user to prevent accidental connecting to SCB network.
GoTo -> certlm.msc 
Ensure all the certificates are removed.
 

GoTo -> certmgr.msc 
Ensure all the certificates are removed.

5	Operational controls

This section defines the mandatory guidelines, do’s and don’ts on the Standalone Computer.
•	Do not download, upload, or install any content that is unrelated to the machine's intended use.
•	Install only licensed third-party software that has been approved by the group.
•	Assure the onboarding of standalone machines into the Group's centralized IT inventory and the current management of the data.
•	Access is recertified through a biannual access review process.
•	Only those with permission can access privileged accounts. 

6	O365 Activation

O365 Activation for SOE / Non-SOE – Get in touch with O365 license team and obtain new standalone license.
Note: Standalone machine shouldn’t use the existing user production license. 
7	Monthly maintenance action process
Being isolated from domain, Standalone Computer is not discovered by in Active Directory or by MECM \ Intune. To keep track and ensure Technical Controls and Operational Controls are adhered, it is recommended to establish monthly maintenance action on the Standalone Computer. Standalone Owner should submit SRM and Support Team verify the monthly checklist.
